# AIchat

Desktop-приложение для общения с моделями искусственного интеллекта через OpenRouter API.

Приложение поддерживает работу с различными AI-моделями, хранение истории диалогов, аналитику использования и безопасную авторизацию по PIN-коду.

---

## Возможности

### Работа с AI-моделями

* Поддержка любых моделей через OpenRouter API
* Контекстные диалоги с сохранением истории сообщений
* Настройка параметров генерации ответов
* Выбор модели в интерфейсе приложения

### Безопасная авторизация

* Проверка API-ключа при первом запуске
* Автоматическая генерация PIN-кода
* Быстрый вход по PIN-коду при последующих запусках
* Сброс API-ключа и PIN-кода
* Хранение данных без необходимости сохранять API-ключ в `.env`

### История чатов

* Автоматическое сохранение диалогов
* Просмотр предыдущих бесед
* Быстрый доступ к истории сообщений

### Аналитика

* Статистика использования моделей
* Подсчет количества запросов
* Мониторинг потребления токенов
* Сбор информации о работе приложения

### Системные функции

* Кэширование данных для повышения производительности
* Подробное логирование
* Мониторинг системных ресурсов
* Обработка ошибок и диагностика


### 3. Активация окружения

**Windows**

```bash
venv\Scripts\activate
```

### 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 5. Настройка переменных окружения

Скопируйте файл:

```bash
envexample
```

в

```bash
.env
```

Пример содержимого:

```env
BASE_URL=https://openrouter.ai/api/v1
DEBUG=False
LOG_LEVEL=INFO
MAX_TOKENS=1000
TEMPERATURE=0.7
```

# Структура проекта

Ниже представлена основная структура проекта и назначение каждого каталога и файла:

* **src/** — исходный код приложения.

  * **api/** — модули для работы с внешними API.

    * `openrouter.py` — взаимодействие с OpenRouter API, отправка запросов и получение ответов от AI-моделей.
  * **ui/** — пользовательский интерфейс приложения.

    * `components.py` — основные элементы интерфейса.
    * `styles.py` — стили и оформление приложения.
  * **utils/** — вспомогательные модули и сервисы.

    * `analytics.py` — сбор статистики использования приложения и моделей.
    * `cache.py` — кэширование данных и истории запросов.
    * `logger.py` — система логирования событий и ошибок.
    * `monitor.py` — мониторинг состояния приложения и системных ресурсов.
  * `main.py` — точка входа приложения.

* **envexample** — пример файла конфигурации с переменными окружения.

* **requirements.txt** — список зависимостей Python.

* **build.py** — скрипт сборки исполняемого файла.

* **INSTALL.md** — подробная инструкция по установке.

* **README.md** — основная документация проекта.

* **.gitignore** — список файлов и каталогов, исключённых из Git.
